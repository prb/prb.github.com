In the spirit of thoroughly reading and implementing specifications, Elliotte Rusty Harold announced XOM 1.0, his

"[...]tree-based API for processing XML with Java. XOM emphasizes correctness, ease-of-use, and performance, in that order."

Elliotte had it already, but anyone who thinks about spell-checking comments gets my respect...

As for reasons to check out XOM, it separates attributes from namespace declarations, and it provides a direct toXML() method for serializing Nodes.
