In response to an earlier posting about Polyphonic C#, Stewart Itzstein pointed me to some of his work on a similar extension to Java. (Itzstein's work predates Polyphonic C#.)

The idea is to combine the join calculus with Java to get a new language that has convenient syntax for concurrent programming. For example:

public class OnePlaceBuffer {
  public Object get() & put(Object o) {
    return o;
  }
}

would have the behavior that the get() method would block until put(Object o) is called, and the get() caller would receive o.
